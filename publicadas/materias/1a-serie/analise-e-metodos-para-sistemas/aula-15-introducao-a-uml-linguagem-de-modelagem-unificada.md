# Aula 15 — Introdução à UML — Linguagem de Modelagem Unificada

Nesta aula, a turma vai conhecer a UML como linguagem padrão de modelagem, entendendo sua função na comunicação entre analistas, desenvolvedores e stakeholders.

## Abertura

:::objetivo
Ao final da aula, a turma deverá: compreender o que é a UML e qual é a sua finalidade no desenvolvimento de software; identificar os principais tipos de diagramas UML; aplicar a UML na representação de aspectos básicos de um sistema.
:::

## Desenvolvimento

:::conceito
A UML (Unified Modeling Language) é uma linguagem gráfica padrão para visualizar, especificar, construir e documentar artefatos de um sistema de software. Ela não é uma metodologia de desenvolvimento, mas sim um conjunto de notações que permite representar diferentes aspectos de um sistema — desde sua estrutura estática até o comportamento dinâmico. A UML é usada por desenvolvedores e analistas em todo o mundo porque oferece uma forma padronizada e precisa de comunicar ideias dentro da equipe e com os stakeholders.
:::

## Conexão com a prática profissional

Em projetos de software, a UML ajuda equipes a documentar, discutir e revisar soluções antes e durante a implementação. Saber interpretar diagramas melhora a comunicação técnica, reduz ambiguidades e apoia decisões de análise, projeto e manutenção.

## Questões de fixação

:::questao Qual é a principal função da UML no desenvolvimento de software?
a) Definir a linguagem de programação que será usada no projeto.
b) Fornecer notações gráficas padronizadas para representar diferentes aspectos de um sistema. *
c) Substituir completamente a documentação textual do sistema.
d) Gerar automaticamente o código-fonte do software.
> A alternativa correta é a B porque a UML oferece símbolos e representações gráficas que permitem documentar e comunicar a estrutura e o comportamento de um sistema de forma padronizada.
:::

:::questao A UML é uma metodologia de desenvolvimento de software?
a) Sim, porque define um processo completo de desenvolvimento.
b) Sim, porque determina todas as etapas desde o levantamento até a entrega.
c) Não — a UML é uma linguagem de modelagem, não um processo ou metodologia. *
d) Sim, porque é usada em todas as empresas de desenvolvimento de software.
> A alternativa correta é a C porque a UML é exclusivamente uma linguagem de notação gráfica que pode ser usada dentro de qualquer metodologia, seja ela clássica ou ágil.
:::

## Atividade prática

:::exercicio
Diagrama UML de um Sistema Simples. O estudante deverá escolher um sistema simples — como um sistema de biblioteca, aplicativo de tarefas ou loja virtual — e criar um diagrama UML representando algum aspecto desse sistema. A atividade introduz a prática da modelagem visual padronizada.
:::

Na atividade prática, a turma deverá escolher um sistema simples — como uma biblioteca, um aplicativo de tarefas ou uma loja virtual — e criar um diagrama UML que represente um aspecto relevante desse sistema.

O estudante deverá:

- Escolher um sistema simples para modelar.
- Identificar os principais elementos desse sistema (usuários, funcionalidades, componentes).
- Selecionar um tipo de diagrama UML adequado ao que se deseja representar.
- Desenhar o diagrama no caderno ou em ferramenta digital.
- Revisar o diagrama para verificar clareza e coerência com o sistema.

:::importante
Entregar em formato de registro no caderno, folha ou arquivo digital, com nome no padrão **Diagrama_UML_NomeAluno**. A entrega pode ser feita no Google Classroom, Drive ou outro ambiente definido pelo professor.
:::

## Fechamento

:::resumo
- A UML é uma linguagem gráfica padrão para modelar sistemas de software.
- Ela não é uma metodologia: pode ser usada com qualquer abordagem de desenvolvimento.
- Seus principais diagramas incluem: Classes, Casos de Uso, Sequência, Atividades, Estado e Componentes.
:::

Na próxima aula, a turma vai aplicar a UML ao Diagrama de Caso de Uso, focando a visão do sistema pelo ponto de vista do usuário.
