# Aula 05 — Levantamento de Requisitos

Nesta aula, a turma vai compreender por que o levantamento de requisitos é uma etapa decisiva do desenvolvimento de software, distinguindo requisitos funcionais e não funcionais e percebendo como essa definição reduz retrabalho e desalinhamentos com o cliente.

## Abertura

:::objetivo
Ao final da aula, a turma deverá: compreender o conceito de levantamento de requisitos e sua importância no desenvolvimento de software; diferenciar requisitos funcionais de requisitos não funcionais; aplicar o levantamento de requisitos a cenários práticos.
:::

## Desenvolvimento

:::conceito
O levantamento de requisitos é uma das fases mais importantes do desenvolvimento de software. É durante essa etapa que cliente e desenvolvedor definem, juntos, o que exatamente o produto deve fazer e como deve se comportar. Sem essa compreensão clara, o risco de retrabalho, falhas de comunicação e soluções desalinhadas cresce significativamente. Nesta aula, a turma vai aprender a coletar, analisar e classificar requisitos em dois grupos principais: funcionais, que descrevem o que o sistema deve fazer, e não funcionais, que descrevem como o sistema deve se comportar sob certas condições.
:::

## Conexão com a prática profissional

Em projetos reais, o levantamento de requisitos transforma demandas genéricas em combinados claros de trabalho. Saber ouvir o cliente, registrar funcionalidades, identificar restrições e validar entendimentos evita retrabalho, melhora a comunicação da equipe e aumenta as chances de o sistema resolver o problema certo.

## Questões de fixação

:::questao Qual alternativa representa corretamente a diferença entre requisitos funcionais e não funcionais?
a) Requisitos funcionais descrevem como o sistema deve se comportar, e não funcionais descrevem o que ele deve fazer.
b) Requisitos funcionais descrevem o que o sistema deve fazer; requisitos não funcionais descrevem como ele deve se comportar. *
c) Requisitos funcionais e não funcionais são sinônimos com nomenclaturas diferentes.
d) Requisitos não funcionais se aplicam apenas a sistemas web e não a aplicativos móveis.
> A alternativa correta é a B porque requisitos funcionais indicam funcionalidades específicas (o que o sistema faz), enquanto não funcionais estabelecem qualidades como desempenho, segurança e usabilidade (como o sistema se comporta).
:::

:::questao Por que o levantamento de requisitos é considerado uma etapa fundamental no desenvolvimento de software?
a) Porque substitui a necessidade de codificação do sistema.
b) Porque permite que o desenvolvedor comece a programar sem entender o cliente.
c) Porque garante que o produto final atenda às reais necessidades do cliente e reduz retrabalhos. *
d) Porque é a única fase que não pode ser adaptada ao longo do projeto.
> A alternativa correta é a C porque o levantamento de requisitos alinha as expectativas do cliente com o trabalho da equipe, evitando que o sistema seja construído fora do escopo e reduzindo custos com correções posteriores.
:::

## Atividade prática

:::exercicio
Lista de Requisitos — Funcional e Não Funcional. O estudante deverá realizar o levantamento de requisitos para um cenário proposto pelo professor, identificando necessidades do cliente, listando requisitos funcionais e não funcionais e organizando tudo em uma lista clara e estruturada. A atividade simula uma situação real em que compreender corretamente o que o cliente precisa é essencial para o sucesso do projeto.
:::

Na atividade prática, a turma deverá realizar o levantamento de requisitos para um cenário proposto pelo professor, identificando necessidades do cliente, listando requisitos funcionais e não funcionais e organizando tudo em uma lista clara e estruturada.

O estudante deverá:

- Ler o cenário apresentado e identificar o contexto do cliente.
- Discutir em dupla ou grupo quais são as reais necessidades do cliente.
- Listar o máximo de requisitos possíveis.
- Classificar cada requisito em funcional ou não funcional.
- Registrar de forma clara e organizada antes da socialização.

:::importante
Entregar em formato de registro no caderno, folha ou arquivo digital, com nome no padrão **Levantamento_Requisitos_NomeAluno**. A entrega pode ser feita no Google Classroom, Drive ou outro ambiente definido pelo professor.
:::

:::roteiro
Hoje vamos aprender que desenvolver software vai muito além de programar. Antes de escrever qualquer linha de código, é preciso entender o problema, ouvir o cliente e registrar com clareza o que o sistema precisa fazer. O levantamento de requisitos é o momento em que transformamos uma necessidade em algo concreto e documentado. Na atividade prática, vocês vão sentir isso na pele: identificar requisitos, separá-los em funcionais e não funcionais e perceber por que essa organização faz toda a diferença na construção de um software que realmente funciona.
:::

## Fechamento

:::resumo
- Levantamento de requisitos é a fase inicial que define o que o software precisa fazer.
- Requisitos funcionais descrevem funcionalidades; não funcionais descrevem qualidades e restrições.
- A classificação correta dos requisitos evita retrabalhos e melhora a comunicação com o cliente.
:::

Na próxima aula, a turma vai transformar esses requisitos em análise e projeto, entendendo como a documentação orienta a construção da solução.
