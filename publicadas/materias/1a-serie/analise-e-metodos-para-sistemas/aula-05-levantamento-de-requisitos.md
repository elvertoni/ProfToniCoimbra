# Aula 05 — Levantamento de Requisitos

Nesta aula, a turma vai o levantamento de requisitos é uma das fases mais importantes do desenvolvimento de software. é durante essa etapa que cliente e desenvolvedor definem, juntos, o que exatamente o produto deve fazer e como deve se comportar. Sem essa compreensão clara, o risco de retrabalho, falhas de comunicação e soluções desalinhadas cresce significativamente. Nesta aula, a turma vai aprender a coletar, analisar e classificar requisitos em dois grupos principais: funcionais, que descrevem o que o sistema deve fazer, e não funcionais, que descrevem como o sistema deve se comportar sob certas condições.

## Objetivos de aprendizagem

- Compreender o conceito de levantamento de requisitos e sua importância no desenvolvimento de software.
- Diferenciar requisitos funcionais de requisitos não funcionais.
- Aplicar o levantamento de requisitos a cenários práticos.

 <div class="questao" data-idx="q1">
 <p class="questao-num">Questão 1</p>
 <p class="questao-enunciado">Qual alternativa representa corretamente a diferença entre requisitos funcionais e não funcionais?</p>
 <ul class="alternativas">
 <li class="alt" data-letra="A"><span class="alt-badge">A</span> Requisitos funcionais descrevem como o sistema deve se comportar, e não funcionais descrevem o que ele deve fazer.</li>
 <li class="alt" data-letra="B" data-correta="true"><span class="alt-badge">B</span> Requisitos funcionais descrevem o que o sistema deve fazer; requisitos não funcionais descrevem como ele deve se comportar.</li>
 <li class="alt" data-letra="C"><span class="alt-badge">C</span> Requisitos funcionais e não funcionais são sinônimos com nomenclaturas diferentes.</li>
 <li class="alt" data-letra="D"><span class="alt-badge">D</span> Requisitos não funcionais se aplicam apenas a sistemas web e não a aplicativos móveis.</li>
 </ul>
 <div class="gabarito">
 <span class="gab-texto">A alternativa correta é a B porque requisitos funcionais indicam funcionalidades específicas (o que o sistema faz), enquanto não funcionais estabelecem qualidades como desempenho, segurança e usabilidade (como o sistema se comporta).</span>
 </div>

A turma deverá O estudante deverá realizar o levantamento de requisitos para um cenário proposto pelo professor, identificando necessidades do cliente, listando requisitos funcionais e não funcionais e organizando tudo em uma lista clara e estruturada. A atividade simula uma situação real em que compreender corretamente o que o cliente precisa é essencial para o sucesso do projeto.

O estudante deverá:

   - Ler o cenário apresentado e identificar o contexto do cliente.
   - Discutir em dupla ou grupo quais são as reais necessidades do cliente.
   - Listar o máximo de requisitos possíveis.
   - Classificar cada requisito em funcional ou não funcional.
   - Registrar de forma clara e organizada antes da socialização.
