# Aula 09 — Análise de Requisitos Inicial — Parte I

Nesta aula, a turma vai todo projeto de software começa com uma necessidade. mas transformar essa necessidade em algo que pode ser construído exige um processo organizado de análise de requisitos. A primeira fase desse processo envolve compreender quem são os stakeholders — todas as partes interessadas no projeto —, entender suas expectativas e começar a delimitar o escopo: o que o sistema vai incluir e, igualmente importante, o que ele não vai incluir. Sem esse trabalho inicial, o projeto perde direção e o risco de construções desalinhadas aumenta significativamente.

## Objetivos de aprendizagem

- Compreender como se inicia o processo de levantamento de requisitos de um sistema.
- Identificar stakeholders e suas expectativas no contexto de um projeto.
- Definir o escopo inicial de um projeto de software.

## Questões de fixação

 <div class="questao" data-idx="q1">
 <p class="questao-num">Questão 1</p>
 <p class="questao-enunciado">O que são stakeholders em um projeto de desenvolvimento de software?</p>
 <ul class="alternativas">
 <li class="alt" data-letra="A"><span class="alt-badge">A</span> São apenas os desenvolvedores que escrevem o código do sistema.</li>
 <li class="alt" data-letra="B" data-correta="true"><span class="alt-badge">B</span> São todas as partes interessadas no projeto, incluindo cliente, usuários finais, gerentes e equipe técnica.</li>
 <li class="alt" data-letra="C"><span class="alt-badge">C</span> São exclusivamente os clientes que contrataram o desenvolvimento do sistema.</li>
 <li class="alt" data-letra="D"><span class="alt-badge">D</span> São os únicos responsáveis pela codificação e pelos testes do software.</li>
 </ul>
 <div class="gabarito">
 <span class="gab-texto">A alternativa correta é a B porque stakeholders incluem qualquer pessoa ou grupo que pode afetar ou é afetado pelo sistema, não apenas quem desenvolve ou quem paga.</span>
 </div>
</div>


 <div class="questao" data-idx="q2">
 <p class="questao-num">Questão 2</p>
 <p class="questao-enunciado">Por que definir o escopo é uma etapa fundamental no início de um projeto de software?</p>
 <ul class="alternativas">
 <li class="alt" data-letra="A"><span class="alt-badge">A</span> Porque o escopo define exatamente todo o código que será escrito.</li>
 <li class="alt" data-letra="B"><span class="alt-badge">B</span> Porque elimina completamente a possibilidade de mudanças durante o projeto.</li>
 <li class="alt" data-letra="C" data-correta="true"><span class="alt-badge">C</span> Porque delimita o trabalho necessário e evita que o projeto perca o foco ao longo do tempo.</li>
 <li class="alt" data-letra="D"><span class="alt-badge">D</span> Porque o escopo substitui a necessidade de levantar requisitos com o cliente.</li>
 </ul>
 <div class="gabarito">
 <span class="gab-texto">A alternativa correta é a C porque o escopo define os limites do projeto, alinhando expectativas e evitando que a equipe se disperse com funcionalidades fora do planejado.</span>
 </div>
</div>

A turma deverá O estudante deverá realizar a aplicação inicial do levantamento de requisitos para um sistema de gerenciamento de estoque, identificando stakeholders, suas necessidades e o escopo inicial do projeto. A atividade simula a primeira reunião entre analista e cliente.

O estudante deverá:

   - Identificar quem são os possíveis stakeholders do sistema.
   - Registrar as necessidades e expectativas de cada stakeholder.
   - Listar informações que precisam ser esclarecidas na primeira reunião.
   - Definir o escopo inicial: o que está e o que não está incluído.
   - Organizar os registros de forma clara para socialização.
