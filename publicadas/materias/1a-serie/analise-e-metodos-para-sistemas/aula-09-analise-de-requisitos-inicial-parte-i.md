# Aula 09 — Análise de Requisitos Inicial — Parte I

Nesta aula, a turma vai iniciar a análise de requisitos identificando stakeholders, necessidades e limites de escopo, etapa essencial para dar direção ao projeto desde o começo.

## Abertura

:::objetivo
Ao final da aula, a turma deverá: compreender como se inicia o processo de levantamento de requisitos de um sistema; identificar stakeholders e suas expectativas no contexto de um projeto; definir o escopo inicial de um projeto de software.
:::

## Desenvolvimento

:::conceito
Todo projeto de software começa com uma necessidade. Mas transformar essa necessidade em algo que pode ser construído exige um processo organizado de análise de requisitos. A primeira fase desse processo envolve compreender quem são os stakeholders — todas as partes interessadas no projeto —, entender suas expectativas e começar a delimitar o escopo: o que o sistema vai incluir e, igualmente importante, o que ele não vai incluir. Sem esse trabalho inicial, o projeto perde direção e o risco de construções desalinhadas aumenta significativamente.
:::

## Conexão com a prática profissional

Na prática profissional, identificar corretamente as partes interessadas evita decisões baseadas em suposições. Quando o analista reconhece quem influencia o projeto e delimita o escopo inicial, a equipe ganha foco e reduz conflitos sobre o que será entregue.

## Questões de fixação

:::questao O que são stakeholders em um projeto de desenvolvimento de software?
a) São apenas os desenvolvedores que escrevem o código do sistema.
b) São todas as partes interessadas no projeto, incluindo cliente, usuários finais, gerentes e equipe técnica. *
c) São exclusivamente os clientes que contrataram o desenvolvimento do sistema.
d) São os únicos responsáveis pela codificação e pelos testes do software.
> A alternativa correta é a B porque stakeholders incluem qualquer pessoa ou grupo que pode afetar ou é afetado pelo sistema, não apenas quem desenvolve ou quem paga.
:::

:::questao Por que definir o escopo é uma etapa fundamental no início de um projeto de software?
a) Porque o escopo define exatamente todo o código que será escrito.
b) Porque elimina completamente a possibilidade de mudanças durante o projeto.
c) Porque delimita o trabalho necessário e evita que o projeto perca o foco ao longo do tempo. *
d) Porque o escopo substitui a necessidade de levantar requisitos com o cliente.
> A alternativa correta é a C porque o escopo define os limites do projeto, alinhando expectativas e evitando que a equipe se disperse com funcionalidades fora do planejado.
:::

## Atividade prática

:::exercicio
Levantamento Inicial de Requisitos — Identificação de Stakeholders e Escopo. O estudante deverá realizar a aplicação inicial do levantamento de requisitos para um sistema de gerenciamento de estoque, identificando stakeholders, suas necessidades e o escopo inicial do projeto. A atividade simula a primeira reunião entre analista e cliente.
:::

Na atividade prática, a turma deverá realizar a aplicação inicial do levantamento de requisitos para um sistema de gerenciamento de estoque, identificando stakeholders, suas necessidades e o escopo inicial do projeto.

O estudante deverá:

- Identificar quem são os possíveis stakeholders do sistema.
- Registrar as necessidades e expectativas de cada stakeholder.
- Listar informações que precisam ser esclarecidas na primeira reunião.
- Definir o escopo inicial: o que está e o que não está incluído.
- Organizar os registros de forma clara para socialização.

:::importante
Entregar em formato de registro no caderno, folha ou arquivo digital, com nome no padrão **Levantamento_Inicial_Requisitos_NomeAluno**. A entrega pode ser feita no Google Classroom, Drive ou outro ambiente definido pelo professor.
:::

:::roteiro
Toda conversa entre cliente e desenvolvedor começa com perguntas. Quem vai usar o sistema? O que essa pessoa precisa de verdade? O que está fora do escopo? Hoje vamos exercer exatamente isso: identificar stakeholders, ouvir suas necessidades e começar a delimitar o escopo de um projeto. Na atividade prática, vocês vão fazer esse papel de analista, coletando informações, organizando ideias e entendendo que as primeiras perguntas são as mais importantes de todas.
:::

## Fechamento

:::resumo
- O levantamento de requisitos começa com a identificação das partes interessadas e suas expectativas.
- Definir o escopo inicial delimita o trabalho e evita perda de foco ao longo do projeto.
- A primeira fase de análise de requisitos é determinante para o sucesso do projeto.
:::

Na próxima aula, a turma vai aprofundar essa análise, explorando perguntas mais precisas e critérios de viabilidade.
