# Aula 16 — UML — Diagrama de Caso de Uso e Elementos Fundamentais

Nesta aula, a turma vai aplicar a UML para representar funcionalidades do ponto de vista do usuário por meio do Diagrama de Caso de Uso, identificando atores, casos de uso e relações básicas.

## Objetivos de aprendizagem

- Identificar atores, casos de uso e relacionamentos básicos em diagramas UML.
- Aplicar o Diagrama de Caso de Uso para representar funcionalidades de um sistema.
- Relacionar a visão do usuário com a modelagem inicial das funcionalidades.

## Desenvolvimento

Entre os diagramas da UML, o Diagrama de Caso de Uso é um dos mais acessíveis para iniciar a modelagem. Ele mostra quem interage com o sistema e quais funcionalidades precisam ser disponibilizadas, ajudando a transformar requisitos em uma visão externa organizada.

Seus elementos fundamentais incluem atores, casos de uso e relacionamentos, que permitem representar de forma simples o escopo funcional de um sistema antes de detalhar sua estrutura interna. Por isso, esse diagrama costuma aparecer nas etapas iniciais de análise e conversa diretamente com o levantamento de requisitos.

## Conexão com a prática profissional

No cotidiano de análise de sistemas, diagramas de caso de uso ajudam a alinhar expectativas com clientes e usuários antes de decisões mais técnicas. Eles são úteis para discutir funcionalidades, delimitar escopo e tornar visível o que o sistema realmente precisa oferecer.

## Questões de fixação

 <div class="questao" data-idx="q1">
 <p class="questao-num">Questão 1</p>
 <p class="questao-enunciado">No Diagrama de Caso de Uso da UML, o que representa um ator?</p>
 <ul class="alternativas">
 <li class="alt" data-letra="A"><span class="alt-badge">A</span> Uma funcionalidade específica que o sistema deve realizar.</li>
 <li class="alt" data-letra="B" data-correta="true"><span class="alt-badge">B</span> Um usuário externo ou outro sistema que interage com o software.</li>
 <li class="alt" data-letra="C"><span class="alt-badge">C</span> Uma classe de objetos dentro do sistema.</li>
 <li class="alt" data-letra="D"><span class="alt-badge">D</span> Uma conexão entre dois casos de uso do sistema.</li>
 </ul>
 <div class="gabarito">
 <span class="gab-texto">A alternativa correta é a B porque um ator representa qualquer elemento externo que interage com o sistema, podendo ser um usuário, outro sistema ou até um dispositivo.</span>
 </div>
</div>


 <div class="questao" data-idx="q2">
 <p class="questao-num">Questão 2</p>
 <p class="questao-enunciado">Qual é a principal diferença entre o Diagrama de Classes e o Diagrama de Caso de Uso?</p>
 <ul class="alternativas">
 <li class="alt" data-letra="A"><span class="alt-badge">A</span> O Diagrama de Classes mostra funcionalidades e o Diagrama de Caso de Uso mostra a estrutura do sistema.</li>
 <li class="alt" data-letra="B"><span class="alt-badge">B</span> Ambos representam a mesma informação de formas diferentes.</li>
 <li class="alt" data-letra="C" data-correta="true"><span class="alt-badge">C</span> O Diagrama de Caso de Uso foca nas funcionalidades do ponto de vista do usuário; o Diagrama de Classes foca na estrutura e nos relacionamentos entre entidades do sistema.</li>
 <li class="alt" data-letra="D"><span class="alt-badge">D</span> O Diagrama de Caso de Uso é usado apenas por desenvolvedores, e o Diagrama de Classes é usado apenas pelo cliente.</li>
 </ul>
 <div class="gabarito">
 <span class="gab-texto">A alternativa correta é a C porque cada diagrama oferece uma perspectiva distinta: o de caso de uso representa o comportamento externo, e o de classes representa a estrutura interna do sistema.</span>
 </div>
</div>

Na atividade prática, a turma deverá modelar funcionalidades básicas de um sistema de e-commerce utilizando o Diagrama de Caso de Uso, identificando o ator principal e as principais funcionalidades que ele pode realizar.

O estudante deverá:

   - Ler o cenário proposto e identificar o ator principal do sistema.
   - Listar as funcionalidades que o sistema deve oferecer ao usuário.
   - Separar essas funcionalidades em casos de uso distintos.
   - Desenhar o Diagrama de Caso de Uso simples com ator e conexões.
   - Revisar se as conexões entre ator e funcionalidades estão corretas.
