# Aula 16 — UML — Diagrama de Caso de Uso e Elementos Fundamentais

Nesta aula, a turma vai aplicar a UML para representar funcionalidades do ponto de vista do usuário por meio do Diagrama de Caso de Uso, identificando atores, casos de uso e relações básicas.

## Abertura

:::objetivo
Ao final da aula, a turma deverá: identificar atores, casos de uso e relacionamentos básicos em diagramas UML; aplicar o Diagrama de Caso de Uso para representar funcionalidades de um sistema; relacionar a visão do usuário com a modelagem inicial das funcionalidades.
:::

## Desenvolvimento

:::conceito
Entre os diagramas da UML, o Diagrama de Caso de Uso é um dos mais acessíveis para iniciar a modelagem. Ele mostra quem interage com o sistema e quais funcionalidades precisam ser disponibilizadas, ajudando a transformar requisitos em uma visão externa organizada.
:::

Seus elementos fundamentais incluem atores, casos de uso e relacionamentos, que permitem representar de forma simples o escopo funcional de um sistema antes de detalhar sua estrutura interna. Por isso, esse diagrama costuma aparecer nas etapas iniciais de análise e conversa diretamente com o levantamento de requisitos.

## Conexão com a prática profissional

No cotidiano de análise de sistemas, diagramas de caso de uso ajudam a alinhar expectativas com clientes e usuários antes de decisões mais técnicas. Eles são úteis para discutir funcionalidades, delimitar escopo e tornar visível o que o sistema realmente precisa oferecer.

## Questões de fixação

:::questao No Diagrama de Caso de Uso da UML, o que representa um ator?
a) Uma funcionalidade específica que o sistema deve realizar.
b) Um usuário externo ou outro sistema que interage com o software. *
c) Uma classe de objetos dentro do sistema.
d) Uma conexão entre dois casos de uso do sistema.
> A alternativa correta é a B porque um ator representa qualquer elemento externo que interage com o sistema, podendo ser um usuário, outro sistema ou até um dispositivo.
:::

:::questao Qual é a principal diferença entre o Diagrama de Classes e o Diagrama de Caso de Uso?
a) O Diagrama de Classes mostra funcionalidades e o Diagrama de Caso de Uso mostra a estrutura do sistema.
b) Ambos representam a mesma informação de formas diferentes.
c) O Diagrama de Caso de Uso foca nas funcionalidades do ponto de vista do usuário; o Diagrama de Classes foca na estrutura e nos relacionamentos entre entidades do sistema. *
d) O Diagrama de Caso de Uso é usado apenas por desenvolvedores, e o Diagrama de Classes é usado apenas pelo cliente.
> A alternativa correta é a C porque cada diagrama oferece uma perspectiva distinta: o de caso de uso representa o comportamento externo, e o de classes representa a estrutura interna do sistema.
:::

## Atividade prática

:::exercicio
Diagrama de Caso de Uso — Sistema de E-commerce. O estudante deverá modelar funcionalidades básicas de um sistema de e-commerce utilizando o Diagrama de Caso de Uso, identificando o ator principal e as principais funcionalidades que ele pode realizar no sistema. A atividade treina a representação de interações usuário-sistema de forma visual e organizada.
:::

Na atividade prática, a turma deverá modelar funcionalidades básicas de um sistema de e-commerce utilizando o Diagrama de Caso de Uso, identificando o ator principal e as principais funcionalidades que ele pode realizar.

O estudante deverá:

- Ler o cenário proposto e identificar o ator principal do sistema.
- Listar as funcionalidades que o sistema deve oferecer ao usuário.
- Separar essas funcionalidades em casos de uso distintos.
- Desenhar o Diagrama de Caso de Uso simples com ator e conexões.
- Revisar se as conexões entre ator e funcionalidades estão corretas.

:::importante
Entregar em formato de registro no caderno, folha ou arquivo digital, com nome no padrão **CasoDeUso_UML_NomeAluno**. A entrega pode ser feita no Google Classroom, Drive ou outro ambiente definido pelo professor.
:::

:::roteiro
Hoje vamos aprofundar o estudo da UML focando em um dos diagramas mais utilizados no mercado: o Diagrama de Caso de Uso. Ele é o ponto de partida de muitas análises porque mostra, de forma simples e direta, o que o sistema precisa fazer do ponto de vista de quem vai usá-lo. Vamos também entender como ele se relaciona com os outros diagramas e por que essa linguagem visual é tão valorizada no mercado de trabalho.

## Fechamento

:::resumo
- O Diagrama de Caso de Uso representa as funcionalidades de um sistema do ponto de vista do usuário.
- Atores são elementos externos que interagem com o sistema; casos de uso são as funcionalidades que ele oferece.
- Diferentes diagramas UML oferecem perspectivas complementares sobre o mesmo sistema.
:::

Na próxima aula, a turma vai complementar essa visão funcional com o Diagrama de Classes, relacionando funcionalidades e estrutura.
