# Aula 10 — Análise de Requisitos Inicial — Parte II

Nesta aula, a turma vai aprofundar a análise de requisitos explorando técnicas de entrevista, formulação de perguntas e análise de viabilidade para verificar se a solução proposta faz sentido no contexto do projeto.

## Abertura

:::objetivo
Ao final da aula, a turma deverá: compreender como aprofundar o levantamento de requisitos com perguntas adequadas; diferenciar requisitos funcionais e não funcionais na prática; realizar uma análise inicial de viabilidade da solução proposta.
:::

## Desenvolvimento

:::conceito
Se a primeira parte do levantamento de requisitos trata de entender com quem estamos lidando e o que o cliente deseja, esta segunda parte vai mais fundo: como o analista faz as perguntas certas para extrair os requisitos verdadeiros, como identificar as partes interessadas com precisão e, finalmente, como avaliar se o que está sendo solicitado é realmente viável. A análise de viabilidade responde perguntas essenciais: o problema está bem definido? Existe um caminho real para resolvê-lo com os recursos disponíveis? Sem essa checagem, projetos podem avançar em direções impraticáveis.
:::

## Conexão com a prática profissional

Analistas de sistemas precisam investigar antes de decidir. Formular boas perguntas, confirmar quem são as partes interessadas e testar a viabilidade técnica, econômica e operacional da solução são práticas que evitam projetos impraticáveis e melhoram a qualidade das decisões.

## Questões de fixação

:::questao Qual é o papel da análise de viabilidade no início de um projeto de software?
a) Determinar exatamente quanto tempo cada desenvolvedor vai trabalhar no projeto.
b) Avaliar se a solução proposta é prática e executável antes de comprometer recursos. *
c) Substituir a necessidade de validar os requisitos com o cliente.
d) Definir a linguagem de programação que será utilizada no sistema.
> A alternativa correta é a B porque a análise de viabilidade verifica se o projeto faz sentido do ponto de vista técnico, econômico e operacional antes de ser aprovado para desenvolvimento.
:::

:::questao Por que fazer as perguntas certas é fundamental no levantamento de requisitos?
a) Porque elimina a necessidade de documentar os requisitos.
b) Porque substitui a etapa de análise de viabilidade.
c) Porque boas perguntas extraem os requisitos que realmente importam e evitam que funcionalidades desnecessárias sejam incluídas. *
d) Porque as perguntas certas garantem que o código será escrito sem erros.
> A alternativa correta é a C porque a qualidade das perguntas determina diretamente a qualidade do entendimento sobre o problema, e perguntas precisas evitam requisitos vagos ou desnecessários.
:::

## Atividade prática

:::exercicio
Levantamento Estruturado de Requisitos com Análise de Viabilidade. O estudante deverá aprofundar o levantamento de requisitos para um sistema de gestão de projetos, identificando stakeholders, registrando requisitos funcionais e não funcionais e realizando uma análise inicial de viabilidade. A atividade simula a segunda etapa do trabalho do analista de requisitos.
:::

Na atividade prática, a turma deverá aprofundar o levantamento de requisitos para um sistema de gestão de projetos, identificando stakeholders, registrando requisitos funcionais e não funcionais e realizando uma análise inicial de viabilidade.

O estudante deverá:

- Listar perguntas que ajudem o cliente a esclarecer suas necessidades.
- Identificar as partes interessadas no sistema.
- Registrar os requisitos funcionais do sistema.
- Registrar os requisitos não funcionais (desempenho, segurança, usabilidade).
- Analisar se a solução é viável considerando o problema apresentado.

:::

:::roteiro
Hoje vamos aprofundar o trabalho que começamos na aula passada. Não basta saber que existe um problema — é preciso fazer as perguntas certas para entendê-lo de verdade. Também vamos aprender a separar o que é funcional do que é não funcional e, mais importante, a se perguntar: isso aqui é realmente viável? Antes de se comprometer com qualquer solução, o bom analista verifica se ela faz sentido. Na atividade prática, vocês vão experimentar esse papel de forma direta.
:::

## Fechamento

:::resumo
- Perguntas bem formuladas extraem requisitos mais precisos e evitam mal-entendidos.
- Requisitos funcionais e não funcionais se complementam na descrição completa de um sistema.
- A análise de viabilidade evita que o projeto siga por um caminho impraticável.
:::

Na próxima aula, a turma vai comparar essa visão analítica com o modelo clássico de desenvolvimento de software, entendendo como as fases se organizam.
