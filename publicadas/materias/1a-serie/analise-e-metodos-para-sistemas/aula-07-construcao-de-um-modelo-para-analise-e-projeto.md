# Aula 07 — Construção de um Modelo para Análise e Projeto

Nesta aula, a turma vai compreender como um modelo de análise e projeto representa um sistema antes da implementação, usando abstração, feedback e refinamento para organizar ideias com mais clareza.

## Abertura

:::objetivo
Ao final da aula, a turma deverá: compreender o que é um modelo de análise e projeto de sistemas; identificar os principais componentes de um modelo de software; construir um modelo inicial de sistema utilizando representações visuais.
:::

## Desenvolvimento

:::conceito
Um modelo de análise e projeto é uma representação abstrata e visual de um sistema que ainda será desenvolvido. Ele permite que a equipe e os stakeholders compreendam a estrutura, as funcionalidades e as interações do futuro software antes mesmo de uma única linha de código ser escrita. A construção desse modelo é um processo colaborativo e iterativo: quanto mais feedbacks são recebidos, mais preciso e útil ele se torna. Técnicas como brainstorming e prototipação são amplamente utilizadas nessa etapa.
:::

## Conexão com a prática profissional

Modelos são usados no trabalho de analistas e desenvolvedores para discutir soluções antes de programar. Quando a equipe modela o sistema com antecedência, consegue testar hipóteses, alinhar expectativas com stakeholders e corrigir problemas de entendimento mais cedo.

## Questões de fixação

:::questao Qual é a principal finalidade de construir um modelo antes de desenvolver um software?
a) Substituir completamente a fase de codificação.
b) Representar visualmente a estrutura e o comportamento do sistema para validar ideias com a equipe e stakeholders. *
c) Garantir que o código-fonte seja escrito mais rapidamente.
d) Definir exclusivamente a aparência visual da interface do sistema.
> A alternativa correta é a B porque o modelo permite que todos os envolvidos visualizem e validem a solução antes do desenvolvimento, evitando erros caros e mal-entendidos.
:::

:::questao Qual técnica é commonly used para gerar ideias e soluções de forma coletiva durante a construção de um modelo de sistema?
a) Codificação em pares.
b) Teste de software.
c) Brainstorming. *
d) Manutenção preventiva.
> A alternativa correta é a C porque o brainstorming é uma técnica colaborativa que stimulationa a geração livre de ideias, permitindo que a equipe construa modelos mais ricos e completos.
:::

## Atividade prática

:::exercicio
Modelo Inicial de Sistema de Gerenciamento de Tarefas. O estudante deverá construir um modelo inicial de sistema para um aplicativo de gerenciamento de tarefas, identificando usuários, funcionalidades principais e a organização lógica entre eles. A atividade desenvolve a capacidade de estruturar e representar um sistema antes da implementação.
:::

Na atividade prática, a turma deverá construir um modelo inicial de sistema para um aplicativo de gerenciamento de tarefas, identificando usuários, funcionalidades principais e a organização lógica entre eles.

O estudante deverá:

- Identificar quem são os usuários do sistema.
- Listar as principais funcionalidades do aplicativo de tarefas.
- Organizar essas funcionalidades em um modelo visual simples.
- Relacionar usuário e funcionalidades (ex.: usuário cria tarefa, edita tarefa).
- Revisar o modelo para verificar clareza e lógica.

:::importante
Entregar em formato de registro no caderno, folha ou arquivo digital, com nome no padrão **Modelo_Sistema_Tarefas_NomeAluno**. A entrega pode ser feita no Google Classroom, Drive ou outro ambiente definido pelo professor.
:::

:::roteiro
Hoje vamos entender que construir software não começa com código — começa com um modelo. Um modelo é como uma planta de uma casa: antes de erguer paredes, é preciso saber como os cômodos se organizam, como as pessoas vão circular e onde cada coisa faz mais sentido. Na atividade prática, vocês vão criar o modelo de um aplicativo de tarefas, pensando em quem usa, o que ele faz e como as partes se conectam. Isso é análise e projeto de verdade.
:::

## Fechamento

:::resumo
- Um modelo de sistema representa visualmente a estrutura e funcionalidades antes da implementação.
- A construção de modelos é iterativa e se beneficia de técnicas colaborativas como brainstorming.
- Prototipar permite validar ideias cedo, evitando retrabalhos no desenvolvimento.
:::

Na próxima aula, a turma vai avançar para a priorização e a validação de requisitos, refinando o que realmente deve entrar na solução.
