# Aula 12 — Modelos de Processos de Software

Nesta aula, a turma vai comparar diferentes modelos de processo de software, percebendo que a escolha entre cascata, ágil e iterativo-incremental depende do contexto do projeto.

## Abertura

:::objetivo
Ao final da aula, a turma deverá: compreender o que são modelos de processos de desenvolvimento de software; diferenciar os modelos cascata, ágil e iterativo-incremental; identificar em quais situações cada modelo é mais adequado.
:::

## Desenvolvimento

:::conceito
Não existe um modelo de desenvolvimento de software que funcione igual para todos os projetos. A escolha da abordagem depende de fatores como a estabilidade dos requisitos, o tamanho da equipe, o prazo disponível e o nível de incerteza do projeto. O modelo cascata funciona bem quando os requisitos são claros desde o início. O desenvolvimento ágil brilha em projetos que precisam se adaptar a mudanças frequentes. Já o modelo iterativo-incremental combina elementos dos dois, permitindo entregas parciais e ajustes ao longo do tempo.
:::

## Conexão com a prática profissional

No mercado, não existe um processo universal. Equipes escolhem modelos diferentes conforme estabilidade dos requisitos, urgência, riscos e frequência de mudanças. Saber justificar essa escolha é parte do trabalho de quem analisa, planeja e conduz projetos de software.

## Questões de fixação

:::questao Em qual situação o modelo cascata (Waterfall) é considerado mais adequado?
a) Projetos com requisitos que mudam constantemente ao longo do desenvolvimento.
b) Projetos com requisitos estáveis e bem definidos desde o início. *
c) Projetos que precisam de entrega contínua de valor ao cliente.
d) Projetos com equipes auto-organizadas e multidisciplinares.
> A alternativa correta é a B porque o modelo cascata exige que cada fase seja concluída antes de avançar, o que funciona bem quando os requisitos são conhecidos e pouco propensos a mudanças.
:::

:::questao O que diferencia o desenvolvimento ágil do modelo cascata?
a) O ágil exige que todos os requisitos sejam definidos e documentados antes do início do projeto.
b) O ágil não utiliza nenhuma forma de planejamento ou organização.
c) O ágil trabalha em pequenas iterações com entregas incrementais, permitindo adaptação contínua a mudanças. *
d) O cascata e o ágil são essencialmente o mesmo modelo com nomes diferentes.
> A alternativa correta é a C porque o desenvolvimento ágil se organiza em ciclos curtos (sprints) com entregas frequentes, o que permite responder a mudanças de requisitos com muito mais flexibilidade do que o modelo sequencial do cascata.
:::

## Atividade prática

:::exercicio
Análise e Escolha de Modelo de Processo. O estudante deverá analisar um cenário de projeto de software e escolher o modelo de processo mais adequado — cascata, ágil ou iterativo-incremental — justificando tecnicamente sua decisão. A atividade desenvolve a capacidade de tomada de decisão fundamentada com base nas características de cada projeto.
:::

Na atividade prática, a turma deverá analisar um cenário de projeto de software e escolher o modelo de processo mais adequado — cascata, ágil ou iterativo-incremental — justificando tecnicamente sua decisão.

O estudante deverá:

- Ler o cenário apresentado e identificar suas características (complexidade, mudanças, prazo, equipe).
- Comparar essas características com os modelos estudados.
- Escolher o modelo mais adequado e justificar a escolha com argumentos claros.
- Registrar a análise de forma organizada.
- Preparar-se para explicar a decisão à turma.

:::importante
Entregar em formato de registro no caderno, folha ou arquivo digital, com nome no padrão **Escolha_Modelo_Processo_NomeAluno**. A entrega pode ser feita no Google Classroom, Drive ou outro ambiente definido pelo professor.
:::

:::roteiro
Hoje vamos comparar três formas diferentes de organizar um projeto de software. Cada uma tem seu momento certo de uso, e saber escolher é uma habilidade que faz diferença na carreira de qualquer profissional de tecnologia. Não existe modelo melhor de forma absoluta — existe modelo mais adequado para cada situação. Na atividade prática, vocês vão analisar cenários reais e decidir qual abordagem usar, justificando a escolha como um verdadeiro profissional da área.
:::

## Fechamento

:::resumo
- Modelos de processo são abordagens estruturadas para organizar o desenvolvimento de software.
- O modelo cascata é sequencial e funciona bem para requisitos estáveis; o ágil é iterativo e se adapta a mudanças.
- A escolha do modelo certo depende das características específicas de cada projeto.
:::
