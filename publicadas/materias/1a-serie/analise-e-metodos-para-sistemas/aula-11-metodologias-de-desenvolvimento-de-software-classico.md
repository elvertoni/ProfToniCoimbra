# Aula 11 — Modelo Clássico de Desenvolvimento de Software

Nesta aula, a turma vai compreender o modelo clássico de desenvolvimento de software, em que as fases se organizam de forma sequencial e cada etapa prepara a próxima.

## Abertura

:::objetivo
Ao final da aula, a turma deverá: compreender o ciclo de vida clássico de desenvolvimento de software; identificar as principais fases do modelo clássico: Levantamento, Análise, Projeto, Codificação, Testes e Manutenção; aplicar o modelo clássico na organização de um projeto de software.
:::

## Desenvolvimento

:::conceito
O ciclo de vida clássico de desenvolvimento de software funciona como um roteiro bem definido: cada fase sucede a anterior em uma sequência lógica e necessária. Essa abordagem, também conhecida como tradicional ou cascata, foi por décadas o padrão da indústria. Cada fase contempla um conjunto de ações que vão desde o diagnóstico do problema até os testes finais antes da entrega. Compreender esse ciclo é fundamental para qualquer profissional de tecnologia, mesmo que hoje outras abordagens também coexistam no mercado.
:::

## Conexão com a prática profissional

Mesmo com a popularização das abordagens ágeis, o modelo clássico continua importante para entender projetos com escopo estável, forte documentação e etapas bem definidas. Conhecer essa lógica ajuda o profissional a reconhecer quando a previsibilidade e o controle são prioritários.

## Questões de fixação

:::questao Quais são as fases do modelo clássico de desenvolvimento de software na ordem correta?
a) Codificação, Testes, Projeto, Análise, Levantamento e Manutenção.
b) Levantamento, Análise, Projeto, Codificação, Testes e Manutenção. *
c) Projeto, Levantamento, Codificação, Análise, Testes e Manutenção.
d) Testes, Levantamento, Manutenção, Projeto, Análise e Codificação.
> A alternativa correta é a B porque o modelo clássico segue uma sequência linear em que cada fase é concluída antes do início da próxima, garantindo organização e controle sobre o progresso do projeto.
:::

:::questao Por que o modelo clássico de desenvolvimento é considerado sequencial?
a) Porque todas as fases são executadas simultaneamente pela equipe.
b) Porque o código é escrito antes de qualquer análise ou planejamento.
c) Porque cada fase sucede a anterior e só é considerada concluída antes de avançar para a próxima. *
d) Porque ele não permite que o cliente acompanhe o andamento do projeto.
> A alternativa correta é a C porque a característica central do modelo clássico é a progressão linear entre fases, o que facilita o controle de cronograma e entregas, mas exige que os requisitos estejam bem definidos desde o início.
:::

## Atividade prática

:::exercicio
Modelo do Ciclo de Desenvolvimento de um Sistema. O estudante deverá aplicar o modelo de processo clássico a um sistema fictício, organizando todas as fases do ciclo de desenvolvimento — desde o levantamento até a manutenção — e descrevendo brevemente cada uma. A atividade desenvolve a compreensão da estrutura sequencial e da importância de cada etapa.
:::

Na atividade prática, a turma deverá aplicar o modelo de processo clássico a um sistema fictício, organizando todas as fases do ciclo de desenvolvimento — do levantamento à manutenção — e descrevendo brevemente cada uma.

O estudante deverá:

- Definir qual sistema será desenvolvido (ex.: biblioteca escolar, agenda digital).
- Descrever a fase de levantamento, identificando informações necessárias.
- Descrever a fase de análise, considerando contexto e viabilidade.
- Planejar as fases de projeto, codificação e testes.
- Descrever a fase de manutenção, prevendo atualizações futuras.

:::importante
Entregar em formato de registro no caderno, folha ou arquivo digital, com nome no padrão **Processo_Classico_NomeAluno**. A entrega pode ser feita no Google Classroom, Drive ou outro ambiente definido pelo professor.
:::

:::roteiro
Hoje vamos conhecer o modelo clássico de desenvolvimento de software — um roteiro que organiza um projeto inteiro em fases bem definidas, uma depois da outra. Pensem como uma linha de produção: cada etapa precisa estar pronta antes que a próxima comece. Isso pode parecer rígido, mas tem uma grande vantagem: controle. Na atividade prática, vocês vão aplicar esse modelo a um sistema fictício, percorrendo cada fase e percebendo como cada uma contribui para a qualidade final do produto.
:::

## Fechamento

:::resumo
- O modelo clássico organiza o desenvolvimento em fases sequenciais: Levantamento, Análise, Projeto, Codificação, Testes e Manutenção.
- Cada fase depende da conclusão da anterior, garantindo controle sobre o progresso do projeto.
- O modelo clássico exige requisitos bem definidos desde o início e funciona bem em projetos com escopo estável.
:::

Na próxima aula, a turma vai comparar o modelo clássico com outros modelos de processo, percebendo quando cada abordagem faz mais sentido.
