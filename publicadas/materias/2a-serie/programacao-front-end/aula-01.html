O que é uma tabela e quando usar

🧠

Tabela HTML

É uma estrutura que organiza dados em formato de grade — linhas e colunas — para facilitar a leitura e comparação de informações.

Tabelas são ideais para exibir dados estruturados como listas de alunos, produtos, notas, horários ou qualquer informação que tenha relação entre categorias. Quando o usuário precisa comparar valores e entender registros de forma organizada, a tabela continua sendo uma das melhores soluções visuais e semânticas.

O erro mais comum dos iniciantes é usar tabelas para criar layout de página. Isso era uma prática antiga da web, mas hoje é considerado má prática. Tabelas devem ser usadas apenas para dados tabulares. Para layout, o correto é usar Flexbox ou CSS Grid.

⚠️

Tabela ≠ Layout

Tabela organiza dados, não páginas. Usar tabela para layout prejudica acessibilidade, manutenção e semântica.

Anatomia da tabela — cada tag explicada

A tabela HTML é formada por um conjunto de tags que trabalham juntas. Entender o papel de cada uma evita código confuso e ajuda a escrever interfaces mais claras.

  • <table> — contêiner principal
  • <thead> — cabeçalho
  • <tbody> — corpo da tabela
  • <tfoot> — rodapé
  • <tr> — linha
  • <th> — cabeçalho
  • <td> — dado

<table>

É o contêiner principal. Tudo que pertence à tabela fica dentro dele.

<thead>

Define o cabeçalho da tabela. Agrupa as linhas que apresentam os títulos das colunas e melhora a semântica do documento.

<tbody>

Agrupa o corpo da tabela, onde ficam os dados reais. Isso separa semanticamente o cabeçalho da parte principal do conteúdo.

<tfoot>

Define o rodapé da tabela. É útil para totais, médias, observações finais ou informações consolidadas.

<tr>

Representa uma linha inteira da tabela. Cada linha reúne células do mesmo registro.

<th>

É a célula de cabeçalho. Pode identificar título de coluna ou de linha. Além do visual, traz significado semântico para a estrutura.

<td>

É a célula de dado comum. É nela que ficam os valores efetivamente mostrados ao usuário.

<table>
  <thead>
    <tr>
      <th>Nome</th>
      <th>Nota</th>
      <th>Situação</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Ana Paula</td>
      <td>8,5</td>
      <td>Aprovada</td>
    </tr>
    <tr>
      <td>Carlos Silva</td>
      <td>6,0</td>
      <td>Aprovado</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td colspan="3">Total de alunos: 2</td>
    </tr>
  </tfoot>
</table>
Nome Nota Situação
Ana Paula 8,5 Aprovada
Carlos Silva 6,0 Aprovado
Total de alunos: 2

Linhas e colunas — a dúvida mais comum

Uma linha é o conjunto horizontal de células que representa um registro completo. Uma coluna é o agrupamento vertical de células da mesma categoria de informação.

Em uma tabela de boletim, por exemplo, cada linha pode representar uma disciplina. Já as colunas representam categorias fixas que se repetem em todas as linhas, como 1º bimestre, 2º bimestre, 3º bimestre, 4º bimestre e média.

O melhor jeito de pensar uma tabela é definir primeiro quais são as colunas. Depois disso, cada nova linha apenas preenche esse padrão. Quando essa lógica é ignorada, a tabela vira uma mistura de dados sem organização.

📌

Regra prática

Se o cabeçalho tem 6 colunas, o restante da tabela precisa seguir esse mesmo padrão, salvo casos específicos de mesclagem com colspan ou rowspan.

Atributos importantes

💡

Mesclando células

Assim como em uma planilha, no HTML isso é feito com os atributos colspan e rowspan.

colspan="N" faz uma célula ocupar N colunas horizontalmente.

rowspan="N" faz uma célula ocupar N linhas verticalmente.

scope é usado em <th> para indicar se o cabeçalho pertence a uma coluna (scope="col") ou a uma linha (scope="row"). Isso melhora bastante a acessibilidade.

<table>
  <thead>
    <tr>
      <th scope="col">Disciplina</th>
      <th scope="col">1º Bimestre</th>
      <th scope="col">2º Bimestre</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Programação</th>
      <td>7,0</td>
      <td>8,5</td>
    </tr>
    <tr>
      <th scope="row">Banco de Dados</th>
      <td>9,0</td>
      <td>6,5</td>
    </tr>
  </tbody>
</table>

CSS básico para uma tabela bem apresentada

No HTML, a estrutura precisa estar correta. No CSS, o objetivo é melhorar a leitura. Não é necessário exagerar: o ideal é resolver legibilidade com poucas regras e um resultado limpo.

As propriedades mais úteis para começar são border, padding, background-color e border-collapse: collapse;. Também vale usar alternância de cor nas linhas para facilitar a leitura.

table {
  width: 100%;
  border-collapse: collapse;
}

th, td {
  border: 1px solid #444;
  padding: 12px;
}

thead {
  background-color: #243244;
  color: white;
}

tbody tr:nth-child(even) {
  background-color: #1a1a1a;
}

Atividade prática — Avaliativa (1,0 ponto)

Agora é hora de aplicar o conteúdo em uma situação próxima da realidade de sistemas escolares.

  1. Crie uma página HTML completa com uma tabela de boletim escolar.
  2. A tabela deve ter pelo menos 4 disciplinas nas linhas.
  3. As colunas devem ser: Disciplina, 1º Bimestre, 2º Bimestre, 3º Bimestre, 4º Bimestre e Média.
  4. Use <thead>, <tbody> e <tfoot> corretamente.
  5. O <tfoot> deve conter uma linha com colspan exibindo o texto Ano letivo: 2025.
  6. Use <th> nos cabeçalhos e scope nas células de título das linhas.
  7. Aplique pelo menos um estilo CSS básico na tabela, como borda, fundo no cabeçalho ou alternância de cores nas linhas.
