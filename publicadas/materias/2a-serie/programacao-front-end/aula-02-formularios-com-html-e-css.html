02O que é um formulário?

Um formulário é a ponte entre o usuário e a aplicação. É por meio dele que o visitante envia dados — como nome, e-mail, senha, preferências — para serem processados pelo sistema. No front-end, o formulário é construído com HTML; no back-end, esses dados são recebidos e tratados.

Regra prática: todo formulário deve ter um propósito claro. Se o usuário não entender o que está preenchendo e por quê, a conversão cai.

Estrutura básica

A tag <form> envolve todos os campos. Dentro dela, usamos <input>, <label>, <textarea>, <select> e <button>:

<form action="/enviar" method="POST">
  <label for="email">E-mail:</label>
  <input type="email" id="email" name="email" required>

  <label for="senha">Senha:</label>
  <input type="password" id="senha" name="senha">

  <button type="submit">Entrar</button>
</form>

Importante: o atributo name é o identificador enviado ao servidor. Sem ele, o campo não é transmitido.

Associando label ao input

A acessibilidade exige que cada campo tenha uma <label> vinculada via atributo for (igual ao id do input). Isso permite que leitores de tela identifiquem o campo e que o usuário clique no texto para focar o input.

<!-- Correto: label associada -->
<label for="nome">Nome completo</label>
<input type="text" id="nome" name="nome">

<!-- Incorreto: label solta -->
<label>Nome completo</label>
<input type="text" name="nome">

Validação com HTML5

O próprio HTML oferece validação sem JavaScript:

  • required — campo obrigatório
  • minlength="8" / maxlength="20" — limita caracteres
  • min="0" / max="100" — limita valores numéricos
  • pattern="[A-Za-z]{3,}" — valida com expressão regular
<input type="text" name="usuario"
       minlength="3" maxlength="15"
       pattern="[a-zA-Z0-9_]+"
       required>

Estilizando com CSS

Um formulário bem estilizado transmite confiança. Veja um exemplo mínimo e funcional:

form {
  display: flex;
  flex-direction: column;
  gap: 16px;
  max-width: 400px;
}

label {
  font-weight: 500;
  color: #e4e4e7;
}

input, textarea, select {
  padding: 12px 14px;
  border: 1px solid #27272a;
  border-radius: 8px;
  background: #18181b;
  color: #fafafa;
  font-size: 1rem;
}

input:focus {
  outline: none;
  border-color: #10b981;
  box-shadow: 0 0 0 3px rgba(16,185,129,0.15);
}

button {
  padding: 12px 20px;
  border: none;
  border-radius: 8px;
  background: #10b981;
  color: #0a0a0a;
  font-weight: 600;
  cursor: pointer;
}

button:hover {
  background: #059669;
}

Dica: use focus para mostrar visualmente qual campo está ativo. Isso melhora a usabilidade e a acessibilidade do formulário.

1

Qual a função do atributo name em um campo de formulário?

  • Definir o tipo de dado que o campo aceita (texto, número, e-mail).
  • Vincular o <label> ao <input> para acessibilidade.
  • Identificar o campo no envio dos dados ao servidor.
  • Aplicar estilos CSS diretamente ao elemento.

Resposta correta: C — O atributo name é o identificador enviado ao servidor. Sem ele, o valor do campo não é transmitido no POST ou GET. O type define o formato (A), o for vincula ao label (B) e CSS usa classes ou IDs (D).

2

Em um grupo de opções onde o usuário deve escolher apenas uma alternativa, qual elemento HTML é o correto?

  • <input type="checkbox">
  • <input type="radio">
  • <input type="select">
  • <textarea>

Resposta correta: B — O type="radio" é projetado para escolha única em um grupo. O checkbox (A) permite múltiplas seleções. select (C) não é um valor válido de type — usa-se a tag <select> separadamente. textarea (D) é para texto longo livre.
